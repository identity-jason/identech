Interfering with Temporal Sets

On one of the environments I'm working in, I need to test a few Temporal sets - simple enough normally, simply make the changes and then then manually execute the FIM_TemporalEventsJob in SQL and wait for the dust to settle - however in this specific system I don't have permissions needed to even see the list of SQL Jobs, never mind arbitrarily execute one.

Fortunately, I do have the rights needed to run all of the individual stored procedures however (or at least can impersonate enough of the FIM infrastructure enough to get away with it!) and by extracting the steps from FIM_TemporalEventsJob have got the following simple SQL - probably missing something vital, but it's enough to keep me moving for the afternoonÂ - and much better than waiting until 1am for all of the changes to be evaluated...

EXECUTE [fim].[TriggerTemporalEvents] @extendedOutput = 1EXECUTE [fim].[MaintainSets]EXECUTE [fim].[MaintainGroups]EXECUTE [fim].[TriggerTemporalEvents] @extendedOutput = 1
