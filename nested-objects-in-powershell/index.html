Nested Objects in Powershell

I’m currently working on automating some activities for an Office365 deployment, and am making use of powershell cmdlets driven through c# – this works a treat most of the time, and aside from a few oddities (usually down to an error between chair & keyboard) drops straight out using code similar to:

//-- Use powershell to retrieve all usersCommand cmd = new Command("get-msoluser");Pipeline pipe = my_run_space.CreatePipeline();pipe.Commands.Add(cmd);IList<PSObject> rv = pipe.Invoke();//-- Now step through the result set and print out some basic detailsforeach (PSObject p in rv){    foreach (PSPropertyInfo ppi in user.Properties)    {        Console.WriteLine("{0}: {1}", ppi.Name, ppi.Value);    }    Console.WriteLine();}

This is fine for most attributes – they’re single valued strings, but a couple of them are multivalued strings – or worse - multivalued objects.

Multivalued strings are simple enough to deal with (I crudely worked it out from the attribute name and went from there) but nested objects were a little more complex – mainly as they were of types that we don’t have easy access to (Microsoft.Online.Automation.*)

Fortunately, this is where .Net’s Reflection capabilities start to shine – by using a generic ICollection and reflecting the properties from there – easy if you know how (or can find a tutorial!)

The ICollection object is used to retrieve a collection of generic objects from Powershell and by then using Reflection.PropertyInfo to identify the property that we need by name (rather than having autocompletion) and again to retrieve the value – our inner loop now becomes:

switch (ppi.Name.ToLower()){    case "licenses":        ICollection collection = (ICollection)ppi.Value;        foreach (object o in collection)        {          //-- Retrieve the AccountSkuID property as this is all that we're actually likely to use            PropertyInfo pi = o.GetType().GetProperty("AccountSkuId");                        Console.WriteLine("AccountSkuId: {0}", (string)pi.GetValue(o, null));        }        break;    default:        Console.WriteLine("{0}: {1}", ppi.Name, ppi.Value);        break;}

While this still won’t work “out-of the-box” (there are some other attributes lurking out there that don’t fall into the default pattern) this method at least allows an arbitrary property to be retrieved from the results of a powershell command whether we have native access to the underlying object-type or not.
