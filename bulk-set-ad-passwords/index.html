Bulk Set AD Passwords

While this isn’t something that you’ll want to be doing too regularly, it’s occasionally useful – especially when you’re dealing with test accounts, migrations (using things like the Live@Edu IMAP Migration tool) or have been having “one of those” days where something as drastic as resetting all passwords starts to look like a good idea.

Fortunately, PowerShell makes this relatively simple and by throwing in a touch of dsquery to get your hands on a user list you can even target selected sub-sets of users providing you can construct a suitable filter to identify them all.

So, after a making sure to load up the AD extensions into PowerShell

import-module ActiveDirectory

All that’s needed after that is something along the lines on…

dsquery user "ou=my users,dc=dev,dc=net" -limit 0 -o samid | % {set-adaccountpassword -Identity $_.Replace('"','') -Reset -NewPassword (ConvertTo-SecureString -AsPlainText "Password123" -Force)}

Which will use dsquery to build a list of all users in the “my users” container, and set their new password to be “Password123” via the set-adaccountpassword cmdlet. 

Obviously, please don’t do this on a production environment without a plan to do something constructive with that password like change it immediately!
